1. GitHubプランの比較 (Free, Team, Enterprise)

GitHubの各プランは、対象となる組織の規模や必要なセキュリティレベルに応じて機能が異なります。

2. 外部コラボレーターの扱い

  • Freeプラン: プライベートリポジトリに無制限に外部コラボレーターを招待でき、料金はかかりません。
  • Teamプラン: 外部コラボレーターをプライベートリポジトリに招待する場合、そのユーザー分のライセンス料金がかかります。パブリックリポジトリへの招待は無料です。

3. コードオーナー & 必須レビュー

レビュープロセスを自動化し、コードの品質を担保する機能です。主にTeamプラン以上で利用できます。

コードオーナー (Code Owners)

リポジトリ内のコードの特定部分に対して、責任を持つ担当者を定義する機能。.github/CODEOWNERSファイルで設定します。

# docsディレクトリ以下のファイルは @docs-team が担当
/docs/    @docs-team

# .jsで終わるファイルは @javascript-devs が担当
*.js      @javascript-devs

必須レビュー (Required Reviews)

プルリクエストがブランチにマージされる前に、指定した条件のレビューが完了していることを強制する「ブランチ保護ルール」の一種です。「コードオーナーからのレビュー」を必須にすることで、より強力なレビュー体制を築けます。

4. 保護されたブランチ

重要なブランチを意図しない変更から守る仕組みです。プランが上位になるほど、より厳格なルールを設定できます。

5. セキュリティと認証機能

SAML SSO (シングルサインオン)

一度のログインで、複数の異なるサービスを利用できるようにする仕組み。主にEnterpriseプランで提供され、組織のID管理システム（IdP）と連携します。Google Workspace (旧 G Suite) をIdPとして利用することも可能です。

高度な監査ログ

「いつ、誰が、何をしたか」をより詳細に記録・追跡する機能。ログの長期保持や、外部監視システムへのリアルタイム転送（ストリーミング）が可能です。

GitHub Advanced Security (GHAS)

開発の早期段階で脆弱性を発見・修正するための統合セキュリティツール群です。Teamプランではアドオンとして購入可能（2025年4月より製品分割）。

1. GitHub Secret Protection: $19 /月（アクティブなコミッター1人あたり）

・主な機能

  • シークレットスキャン (Secret scanning): コード内のAPIキーやパスワードを検出
  • プッシュ保護 (Push protection): 機密情報がリモートリポジトリにプッシュされるのをブロック

2. GitHub Code Security: $30 /月（アクティブなコミッター1人あたり）

  • コードスキャン (Code scanning): CodeQLによるコード内の脆弱性スキャン。
  • シークレットスキャン (Secret scanning): APIキーなどの機密情報を検出し、プッシュをブロック。
  • 依存関係レビュー (Dependency review): オープンソースライブラリの脆弱性をチェック。
